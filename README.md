# CardioVision - AI-Based Heart Disease Detection

CardioVision is a web application that uses deep learning to detect heart disease risk from retinal fundus images.

## Features

- 🔐 User authentication (signup/login)
- 🤖 AI-powered heart disease risk prediction
- 🔥 Grad-CAM heatmap visualization
- 📄 PDF medical report generation
- 💾 MongoDB database for user records
- ✨ Premium, high-quality dynamic UI with glassmorphism design

## Technology Stack

**Backend:**
- FastAPI
- PyTorch (EfficientNet-B7) (Integration ready)
- MongoDB (Motor Async)
- JWT Authentication

**Frontend:**
- Vanilla JavaScript
- HTML5/CSS3 (Custom Premium Design)

## Setup Instructions

### 1. Prerequisites

- Python 3.8+
- MongoDB running locally or MongoDB Atlas Account
- Your trained model file: `cardiovision_b7.pth`

### 2. Installation

1. Start by creating a virtual environment:
```bash
python -m venv venv
```

2. Activate the virtual environment
- On Windows: `venv\Scripts\activate`
- On Mac/Linux: `source venv/bin/activate`

3. Install requirements
```bash
pip install -r requirements.txt
```

### 3. Environment Configuration

Check the `.env` file in the root directory. It is pre-configured for local development, but you should update `MONGODB_URL` and `SECRET_KEY` if you are deploying to production.

### 4. Add Model File

Place your trained model file `cardiovision_b7.pth` in the project root directory.

### 5. Start the Application

Start the FastAPI application using uvicorn:
```bash
cd backend
python -m uvicorn app:app --reload --host 127.0.0.1 --port 8000
```

### 6. Access the Frontend

Open `frontend/login.html` directly in your browser or serve it using a simple HTTP server (e.g. `python -m http.server 5500` inside the `frontend` folder).

## API Documentation

The interactive API documentation is automatically generated by FastAPI. Once the server is running, visit:
- Swagger UI: `http://127.0.0.1:8000/docs`
- ReDoc: `http://127.0.0.1:8000/redoc`
